The brief

What you're building

One button that does two jobs: it switches the page between light and dark, and it tells a little story while it does. The sun slides across and turns into a moon, clouds drift away as stars fade in, and the sky cross-fades from afternoon blue to midnight. No libraries — just HTML, CSS, and about ten lines of JavaScript.

Level · beginner-friendly ~30 minutes Skills · transforms, transitions, easing Ships as · a real, publishable component
Step 0

Name the parts before you animate them

Every smooth interaction starts boring: a stack of plain elements. Animation is just these parts changing one or two properties. Here's the whole switch, taken apart.

Track
The rounded pill. Clips everything inside it with overflow:hidden.
Thumb
The circle that slides. It's the sun in light, the moon in dark.
Two skies
A day gradient and a night gradient, stacked — we cross-fade their opacity.
Clouds
White pills with two bumps. They drift at rest, then slide out.
Stars
Tiny dots that fade in and twinkle forever once it's night.
Craters
Three grey circles inside the thumb — transparent on the sun, visible on the moon.
Step 1

The markup

The state lives in one place: a data-theme attribute on the <html> element. Everything else just reacts to it. That single source of truth is what lets the same flip drive the switch and the whole page.

<!-- one attribute controls the whole page -->
<html data-theme="light">

<button class="switch" role="switch">
  <span class="sky day"></span>
  <span class="sky night"></span>
  <span class="stars"> … dots … </span>
  <span class="clouds"> … clouds … </span>
  <span class="thumb">
    <span class="crater"></span> ×3
  </span>
</button>
Step 2

The track, and the slide

The thumb is positioned left at rest. We never animate left — that's slow and janky. Instead we move it with transform: translateX(), which the browser animates on the GPU. I store the travel distance in a CSS variable so the maths stays readable.

.switch {
  --w: 230px; --thumb: 80px; --pad: 10px;
  --travel: calc(var(--w) - var(--thumb) - var(--pad)*2);
  overflow: hidden;            /* clip clouds + stars */
}
.thumb {
  transform: translateX(0);     /* sun, on the left */
}
[data-theme="dark"] .thumb {
  transform: translateX(var(--travel));  /* moon, on the right */
}
Step 3

Turning the sun into a moon

The same circle plays both roles. In light it's a warm radial gradient with a glowing box-shadow. In dark we swap to a pale grey gradient, cool the glow, and fade in three craters that were transparent the whole time. Because both states share the element, the browser tweens smoothly between them.

.thumb {                          /* ☀ sun */
  background: radial-gradient(circle at 35% 32%, #ffe588, #ffb43c);
  box-shadow: 0 0 30px 8px rgba(255,176,60,.6);
}
[data-theme="dark"] .thumb {       /* 🌙 moon */
  background: radial-gradient(circle at 38% 34%, #f7f4ea, #cdc9bc);
  box-shadow: 0 0 26px 5px rgba(190,205,235,.45);
}
.crater { background: transparent; }        /* hidden on sun */
[data-theme="dark"] .crater { background: rgba(168,170,182,.55); }
Step 4

Sky, clouds & stars

The two skies cross-fade by flipping opacity. Clouds slide out with translateX while their opacity drops. Stars do the reverse — fade and scale in. Each lives behind the thumb, so as the moon arrives, the night settles in around it.

.sky.night { opacity: 0; }
[data-theme="dark"] .sky.night { opacity: 1; }   /* cross-fade */

[data-theme="dark"] .clouds {
  transform: translateX(130px); opacity: 0;   /* drift away */
}
.stars { opacity: 0; }
[data-theme="dark"] .stars { opacity: 1; }       /* fade in + twinkle */
Step 5 — the important one

The secret to making it feel natural

Swapping properties is easy. Making it feel alive is the craft, and it comes down to three habits.

01

Easing with a little overshoot

A linear slide looks mechanical. Real objects have momentum — they overshoot a touch, then settle. A cubic-bézier that dips below 0 and rises past 1 gives the thumb exactly that gentle, springy landing.

02

Stagger the timings

Nothing in nature happens all at once. The sky fades over 0.8s; the stars wait 0.15s before arriving; the craters wait 0.18s so the moon "becomes" a moon after it lands. Those tiny offsets are what read as organic.

03

Give it life at rest

Even when no one's touching it, the sun breathes, the clouds drift, and the stars twinkle — looping animations that never stop. Stillness feels dead; gentle motion feels warm.

.thumb {
  /* the springy landing → "natural" */
  transition: transform .75s cubic-bezier(.66, -.32, .28, 1.34);
}
.stars  { transition: opacity .6s ease .15s; }   /* arrive late */
.crater { transition: background .55s ease .18s; } /* later still */

@keyframes breathe {           /* the sun, alive at rest */
  0%,100% { transform: scale(1);    opacity: .7; }
  50%     { transform: scale(1.14); opacity: 1; }
}
⌥ don't skip this

Always respect prefers-reduced-motion. Some people get motion sick. Wrap the big animations so they collapse to a quick fade — the switch still works, it just stops dancing. Accessible is part of the craft.

Step 6

Wire it up — the whole script

This is all the JavaScript there is. Flip the attribute; CSS does the rest. And because the page's colours are stored as variables that key off the same data-theme, one click re-themes everything you're looking at right now.

const root = document.documentElement;
const sw = document.querySelector('.switch');

sw.addEventListener('click', () => {
  const next = root.dataset.theme === 'dark' ? 'light' : 'dark';
  root.dataset.theme = next;                 /* that's it */
  sw.setAttribute('aria-checked', next === 'dark');
});
↗ make it yours

Try these once it works: start from the visitor's system setting with @media (prefers-color-scheme: dark); add a shooting star on the way into night; recolour the sky for a sunset palette; or slow everything down to 1.2s and watch how much more "premium" it suddenly feels.

For the NextWork team

How I'd turn this into a whole series

This switch isn't a one-off — it's a format. A tiny interaction that teaches one idea, documents itself as it's built, and leaves the learner with a real component to publish. That's the NextWork loop exactly: build → document → proof. Here's the template every lesson in the series would follow.

i
One interaction, one concept. The whole switch only teaches transforms & easing. The constraint is what makes it learnable in 30 minutes.
ii
Vanilla only. No frameworks. If a beginner can rebuild it from scratch, it belongs in the series.
iii
Anatomy first. Name every part before animating. Understanding beats copy-paste.
iv
Isolate the craft move. Every lesson has one "secret" section — here, easing + stagger. That's the bit that separates fine from delightful.
v
End on a challenge. A "make it yours" prompt so each learner ships a unique variant — instant proof of work for their portfolio.
